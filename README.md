# Airflow Data Engineering Project: COVID Daily Pipeline

This project is a local data engineering stack built with Apache Airflow, Spark, HDFS, Hive, and PostgreSQL (via Docker Compose). It orchestrates a daily COVID data ingestion and transformation pipeline.

## What This Project Does

The DAG (`covid_daily_dag`) runs a daily workflow:

1. Checks that a daily COVID CSV is available from the source endpoint.
2. Downloads the CSV file to Airflow local storage.
3. Uploads the file to HDFS.
4. Runs Spark processing to calculate country-level aggregates and ratios.
5. Creates a Hive external table if needed and updates table partitions.

## Tech Stack

- Apache Airflow 2.9.1 (LocalExecutor)
- Apache Spark 3.5.3
- Hadoop HDFS (NameNode + DataNode)
- Apache Hive 4.0.0 (Metastore + HiveServer2)
- PostgreSQL 15 (Airflow DB and Hive Metastore DB)
- Python 3.11

## Repository Structure

- `docker-compose.yml` - Full local platform definition (Airflow, HDFS, Hive, Spark, Postgres)
- `Dockerfile` - Custom Airflow image with required provider packages
- `requirements.txt` - Additional Airflow provider requirements
- `dags/covid_daily_dag.py` - Main Airflow DAG
- `dags/scripts/covid_data_processing.py` - Spark transformation job
- `hive/conf/hive-site.xml` - Hive metastore and warehouse config
- `hadoop/config/core-site.xml` - Hadoop core config
- `hadoop/config/hdfs-site.xml` - HDFS config
- `logs/` - Runtime logs generated by Airflow

## Prerequisites

- Docker + Docker Compose installed
- Enough resources for the full stack (Airflow, Spark, Hive, HDFS, Postgres)
- Internet access for downloading source COVID CSV files

## Quick Start

1. Build and initialize Airflow metadata:

```bash
docker compose up airflow-init
```

2. Start all services:

```bash
docker compose up -d
```

3. Open Airflow UI:

- URL: [http://localhost:8080](http://localhost:8080)
- Default credentials from compose init command:
  - username: `admin`
  - password: `admin`

4. Enable DAG `covid_daily_dag` in the Airflow UI and trigger a run.

## Service Endpoints

- Airflow UI: `localhost:8080`
- HDFS NameNode UI: `localhost:9870`
- Hive Metastore (thrift): `localhost:9083`
- HiveServer2: `localhost:10000`
- Spark Master UI: `localhost:8090`
- Spark Master: `spark://localhost:7077`

## Airflow Connections Required

The DAG expects these Airflow connections to exist:

- `covid_api` (HTTP)
  - Host: `https://raw.githubusercontent.com/`
- `hdfs_default` (WebHDFS)
  - Host: `namenode`
  - Port: `9870`
  - Extra (example): `{"use_ssl": false}`
- `spark_conn` (Spark)
  - Host: `spark://spark-master`
  - Port: `7077`
- `hive_conn` (Hive CLI / HiveServer2 depending on operator/hook usage)
  - Host: `hive-server`
  - Port: `10000`
  - Login: optional, depending on your Hive setup

You can create these from Airflow UI: **Admin -> Connections**.

## Data Flow

- Source file pattern:
  - `https://raw.githubusercontent.com/CSSEGISandData/COVID-19/refs/heads/master/csse_covid_19_data/csse_covid_19_daily_reports/<MM/DD/YYYY>.csv`
- Local temporary download path:
  - `/opt/airflow/data/<MM/DD/YYYY>.csv`
- HDFS raw path:
  - `/covid_data/csv/<MM/DD/YYYY>.csv`
- HDFS processed output path:
  - `/covid_data/result/exec_data=<MM/DD/YYYY>/`
- Hive external table location (expected):
  - `/covid_data/results`

## Notes and Current Gaps

- The project is set up as a local learning/lab environment, not production-ready.
- Some values and identifiers in the DAG/script are inconsistent (`exec_data` vs `exec_date`, `result` vs `results`, etc.), which may require code cleanup for reliable end-to-end runs.
- `docker-compose.yml` mounts a PostgreSQL JDBC jar from `./lib/postgresql-42.7.11.jar`; ensure this file exists locally.
- The repository currently includes IDE files, logs, and Python cache artifacts that are usually excluded via `.gitignore`.
